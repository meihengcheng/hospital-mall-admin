# 医院商城管理后台 - 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 项目背景
医院商城管理后台是医疗机构数字化转型的关键组成部分，旨在通过线上平台实现药品、医疗器械、保健品等医疗相关商品的管理与销售，提升医院服务效率，优化患者体验。

### 1.2 项目目标
- 构建集"在线诊疗、药品销售、健康管理、供应链协同"于一体的综合性管理平台
- 实现医疗服务全流程数字化管理
- 提升患者就医便捷性，优化医院运营效率
- 确保合规安全，严格遵循医疗行业相关法规

### 1.3 目标用户
- 医院管理员
- 药房管理员
- 药库管理员
- 医生/药师
- 财务人员
- 运营人员

## 2. 功能模块设计

### 2.1 用户管理模块

#### 功能目标
管理商城注册用户及权限，保证用户信息安全。

#### 功能点
| 功能 | 描述 |
|------|------|
| 用户列表查看 | 分页显示注册用户信息（ID、用户名、手机号、实名认证状态、注册时间、状态） |
| 用户详情查看 | 查看用户完整信息、就诊记录、订单历史、健康档案 |
| 用户信息编辑 | 修改用户基本信息、联系方式 |
| 用户状态管理 | 冻结/解冻用户账号、封禁违规用户 |
| 会员等级管理 | 普通会员、银卡会员、金卡会员、VIP会员等级设置 |
| 实名认证审核 | 审核用户实名认证信息（身份证、医保卡） |
| 登录日志查看 | 查看用户登录设备、时间、IP地址 |

---

### 2.2 商品管理模块

#### 功能目标
管理商城商品信息、库存和价格，确保商品合规上架。

#### 功能点
| 功能 | 描述 |
|------|------|
| 商品列表查看 | 分页展示商品（ID、名称、类别、规格、价格、库存、状态、效期） |
| 商品新增 | 添加新商品：名称、描述、价格、库存、图片、批准文号、生产厂家 |
| 商品编辑 | 修改商品信息、价格、库存、效期 |
| 商品上下架 | 控制商品是否可购买 |
| 商品分类管理 | 添加/删除/编辑商品类别（药品、医疗器械、保健品等） |
| 效期管理 | 设置药品有效期，近效期预警（提前3个月、6个月） |
| 资质审核 | 上传药品批准文号、生产许可证、质检报告 |
| 批量操作 | 批量上下架、批量修改价格、批量导入导出 |
| SKU管理 | 多规格商品管理（不同规格、不同价格） |

#### 商品分类体系
- **药品**：处方药、OTC药品、中药饮片、中成药
- **医疗器械**：家用医疗器械、康复辅助器具、医用耗材
- **保健品**：保健食品、营养补充剂
- **健康服务**：体检套餐、康复指导、护理服务

---

### 2.3 订单管理模块

#### 功能目标
管理用户下单、支付、发货、退货全流程。

#### 功能点
| 功能 | 描述 |
|------|------|
| 订单列表查看 | 显示订单号、用户、状态、金额、下单时间、配送方式 |
| 订单状态管理 | 待支付、已支付、待发货、已发货、已完成、已取消、退货中 |
| 订单详情查看 | 商品明细、用户信息、收货地址、处方信息、物流信息 |
| 处方订单处理 | 接收电子处方、药师审核、审核通过/拒绝 |
| 退款管理 | 处理用户申请退款和退货 |
| 订单统计 | 订单总量、完成率、退款率、销售额统计 |
| 订单导出 | 导出订单数据为Excel/PDF |

#### 订单状态流转
```
待支付 → 已支付 → 待发货 → 已发货 → 已完成
  ↓         ↓         ↓         ↓
已取消   退款中    退货中    已取消
```

### 2.4 库存管理模块

#### 功能目标
确保商品库存准确，防止超卖，实现精细化库存管理。

#### 功能点
| 功能 | 描述 |
|------|------|
| 库存查看 | 实时库存显示，多仓库库存管理 |
| 库存预警 | 低库存预警（低于安全库存阈值）、近效期预警 |
| 入库管理 | 采购入库、退货入库、调拨入库 |
| 出库管理 | 销售出库、报损出库、调拨出库 |
| 库存盘点 | 定期/临时盘点，自动生成盘点单，计算盘盈盘亏 |
| 库存调拨 | 不同仓库间库存调拨 |
| 库存调整 | 手动调整库存，记录调整原因 |
| 库存报表 | 库存周转率、滞销商品统计 |

---

### 2.5 处方管理模块

#### 功能目标
对接互联网医院电子处方系统，实现处方合规流转。

#### 功能点
| 功能 | 描述 |
|------|------|
| 处方接收 | 自动接收互联网医院电子处方 |
| 处方审核 | 药师在线审核处方合法性、规范性、适宜性 |
| 审核标准 | 四查十对：查处方、查药品、查配伍禁忌、查用药合理性 |
| 审核结果 | 通过/拒绝，拒绝需注明原因 |
| 处方流转 | 支持院内自提、配送到家两种取药方式 |
| 处方查询 | 按患者、医生、时间等条件查询处方 |
| 处方统计 | 处方量统计、审核通过率统计 |

---

### 2.6 配送管理模块

| 配送统计 | 配送时效统计、配送成功率统计 |

## 3. 非功能性需求

### 3.1 性能需求
- 页面加载时间 < 3秒
- 支持并发用户 1000+
- 数据库查询响应时间 < 500ms

### 3.2 安全需求
- 数据加密传输（HTTPS）
- 敏感数据加密存储
- 访问控制与权限验证
- 操作日志记录
- 防SQL注入、XSS攻击

### 3.3 可用性需求
- 系统可用性 99.9%
- 支持数据备份与恢复
- 支持故障自动切换

### 3.4 兼容性需求
- 支持主流浏览器（Chrome、Firefox、Safari、Edge）
- 响应式设计，支持不同屏幕尺寸
