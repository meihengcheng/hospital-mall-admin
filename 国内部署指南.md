## 🚀 方案一：Gitee Pages（最推荐，完全免费）

Gitee Pages 是国内访问最快的免费静态托管方案。

### 部署步骤

1. **注册/登录 Gitee**
   - 访问 https://gitee.com
   - 注册或登录账号

2. **创建仓库**
   - 点击右上角 "+" → "新建仓库"
   - 仓库名称：`hospital-mall-admin`（或自定义）
   - 设置为**公开仓库**
   - 点击"创建"

3. **上传代码**
   
   在本地项目目录执行：
   ```bash
   cd d:\hospital-mall-admin
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin https://gitee.com/你的用户名/hospital-mall-admin.git
   git push -u origin master
   ```

4. **启用 Gitee Pages**
   - 进入仓库页面
   - 点击顶部菜单 "服务" → "Gitee Pages"
   - 部署分支：选择 `master`
   - 部署目录：填写 `dist`
   - 点击"启动"

5. **等待部署完成**
   - 通常 1-2 分钟即可完成
   - 部署成功后会获得访问地址，类似：
     `https://你的用户名.gitee.io/hospital-mall-admin`

## 🚀 方案二：GitHub Pages + jsdelivr CDN（免费）

如果已有 GitHub 仓库，可以使用 GitHub Pages 配合 jsdelivr CDN 加速。

### 部署步骤

1. **推送到 GitHub**
   ```bash
   git remote add origin https://github.com/你的用户名/hospital-mall-admin.git
   git push -u origin main
   ```

2. **启用 GitHub Pages**
   - 进入仓库 Settings → Pages
   - Source: `Deploy from a branch`
   - Branch: `main` / `dist` 目录
   - 点击 "Save"

3. **使用 jsdelivr 加速访问**
   
   GitHub Pages 地址格式：
   ```
   https://你的用户名.github.io/hospital-mall-admin
   ```
   
   使用 jsdelivr CDN 加速（访问更快）：
   ```
   https://cdn.jsdelivr.net/gh/你的用户名/hospital-mall-admin@gh-pages/
   ```

## 🚀 方案三：腾讯云静态网站托管（推荐，首年免费）

腾讯云 COS 静态网站托管，国内访问速度极快。

### 部署步骤

1. **注册腾讯云账号**
   - 访问 https://cloud.tencent.com
   - 完成实名认证

2. **创建 COS 存储桶**
   - 进入对象存储 COS 控制台
   - 创建存储桶（选择就近地域）
   - 访问权限设为"公有读私有写"

3. **上传 dist 目录**
   - 将本地 `dist` 目录下所有文件上传到存储桶
   - 或使用 COSCMD 工具批量上传

4. **开启静态网站**
   - 进入存储桶"基础配置" → "静态网站"
   - 开启"静态网站"
   - 索引文档：`index.html`
   - 错误文档：`index.html`（用于 SPA 路由）

5. **获取访问地址**
   - 在"静态网站"页面可看到访问域名
   - 可以绑定自定义域名（可选）

## 🚀 方案四：阿里云 OSS 静态托管

阿里云 OSS 静态网站托管，稳定可靠。

### 部署步骤

1. **注册阿里云账号**
   - 访问 https://www.aliyun.com
   - 完成实名认证

2. **创建 OSS Bucket**
   - 进入 OSS 控制台
   - 创建 Bucket（选择就近地域）
   - 读写权限：公共读

3. **上传文件**
   - 将 `dist` 目录内容上传到 Bucket
   - 或使用 OSSBrowser 工具

4. **配置静态网站**
   - 进入 Bucket"数据管理" → "静态页面"
   - 设置默认首页：`index.html`
   - 设置默认 404 页：`index.html`

5. **获取访问地址**
   - 在 Bucket 概览页面可看到访问域名

## 📂 项目配置说明

### Mock 模式开关

在 `src/utils/request.ts` 中可切换模式：

```typescript
// 是否使用 Mock 数据（设为 true 则完全使用前端 Mock，无需后端）
const USE_MOCK = true  // 改为 false 则使用真实后端 API
```

### 构建项目

如需重新构建：
```bash
npm run build
```

构建产物在 `dist` 目录。

## ⚠️ 注意事项

1. **数据持久化**：当前 Mock 数据存储在内存中，刷新页面后修改的数据会重置
2. **生产环境**：如需正式使用，建议连接真实数据库
3. **HTTPS**：以上方案均自动支持 HTTPS
4. **自定义域名**：所有平台都支持绑定自定义域名

## 🆘 常见问题

**Q: Gitee Pages 部署后显示 404？**
A: 检查部署目录是否正确设置为 `dist`，确保已执行 `npm run build`。

**Q: 页面刷新后 404？**
A: 这是 SPA 单页应用的特性，需要配置服务端将所有路由重定向到 `index.html`。

**Q: 如何切换回真实后端？**
A: 修改 `src/utils/request.ts` 中的 `USE_MOCK = false`，然后重新构建。
