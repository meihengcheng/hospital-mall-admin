# 医院商城管理后台 - 部署指南

## 快速部署到 Vercel（推荐）

本项目已配置好 Vercel 部署，只需几分钟即可完成部署！

### 方法一：使用 Vercel 官网部署

1. **将代码推送到 GitHub/GitLab/Bitbucket**
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin <你的仓库地址>
   git push -u origin main
   ```

2. **访问 Vercel 官网**
   - 打开 https://vercel.com
   - 使用 GitHub/GitLab 账号登录

3. **导入项目**
   - 点击 "New Project"
   - 选择刚刚推送的仓库
   - 点击 "Import"

4. **配置项目（通常保持默认即可）**
   - Project Name: 输入项目名称（如：hospital-mall-admin）
   - Framework Preset: Vite（会自动识别）
   - Root Directory: 保持默认
   - Build Command: `npm run build`（会自动识别）
   - Output Directory: `dist`（会自动识别）

5. **点击 "Deploy"**
   - 等待 1-2 分钟
   - 部署完成后会获得一个类似 `https://hospital-mall-admin-xxxxx.vercel.app` 的网址

### 方法二：使用 Vercel CLI 部署

1. **安装 Vercel CLI**
   ```bash
   npm install -g vercel
   ```

2. **登录 Vercel**
   ```bash
   vercel login
   ```

3. **部署项目**
   ```bash
   cd d:\hospital-mall-admin
   vercel
   ```
   - 按照提示回答问题（通常一路回车即可）
   - 第一次部署会询问是否设置为生产环境，选择 `Y`

4. **部署到生产环境**
   ```bash
   vercel --prod
   ```

## 其他部署选项

### 部署到 Netlify

1. 将代码推送到 GitHub
2. 访问 https://app.netlify.com
3. 点击 "Add new site" → "Import an existing project"
4. 选择仓库
5. 配置构建命令：`npm run build`
6. 发布目录：`dist`
7. 点击 "Deploy site"

### 部署到 GitHub Pages

1. 在 `vite.config.ts` 中配置 base 路径
2. 构建项目：`npm run build`
3. 部署 `dist` 目录到 GitHub Pages

### 传统服务器部署（Nginx）

1. 构建项目：
   ```bash
   npm run build
   ```

2. 将 `dist` 目录上传到服务器

3. 配置 Nginx：
   ```nginx
   server {
       listen 80;
       server_name your-domain.com;
       root /path/to/dist;
       index index.html;
       
       location / {
           try_files $uri $uri/ /index.html;
       }
       
       location /api {
           proxy_pass http://localhost:3001;
           proxy_http_version 1.1;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection 'upgrade';
           proxy_set_header Host $host;
           proxy_cache_bypass $http_upgrade;
       }
   }
   ```

## 注意事项

1. **API 数据**：当前使用的是内存中的 Mock 数据，每次部署或重启服务器后数据会重置
2. **生产环境**：如需持久化数据，建议连接真实数据库（MongoDB/MySQL 等）
3. **域名**：Vercel 提供免费域名，也可以绑定自己的域名
4. **HTTPS**：Vercel 自动提供 HTTPS 支持

## 项目结构说明

- `api/` - Vercel Serverless Functions（后端 API）
- `src/` - 前端源代码
- `dist/` - 构建输出目录（构建后生成）
- `vercel.json` - Vercel 配置文件
- `vite.config.ts` - Vite 构建配置
